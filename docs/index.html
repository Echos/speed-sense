Privacy Policy

Last updated: March 28, 2026

SpeedSense does not collect, transmit, or share any personal data. All processing happens locally in your browser.

Data We Do Not Collect

  • No browsing history or visited URLs
  • No audio or video content from any page
  • No personal information of any kind
  • No analytics or usage statistics sent to any server

What Is Stored Locally

SpeedSense stores the following data in chrome.storage.local — on your device only, never transmitted externally:

  • Playback speed settings (per domain)
  • Silence detection threshold and delay
  • Keyboard shortcut assignments
  • Cumulative time-saved counter
  • UI language preference (EN / JA)

This data never leaves your device. You can clear it at any time via the Reset button in the popup, or by removing the extension.

External Requests

The extension popup loads fonts from Google Fonts (fonts.googleapis.com) for display purposes only. No user data is included in this request. If you prefer no external requests, this does not affect the core functionality of the extension.

Permissions Explanation

  • storage — to save your settings locally
  • tabs — to communicate between the popup and the active page; used to open the support page when clicked
  • host_permissions (<all_urls>) — required to inject the content script into any page that contains a video element; no data is read or sent from those pages

Contact

If you have any questions about this policy, please open an issue on GitHub.

プライバシーポリシー

最終更新日: 2026年3月28日

SpeedSense は個人データを一切収集・送信・共有しません。すべての処理はお使いのブラウザ内でローカルに行われます。

収集しないデータ

  • 閲覧履歴・アクセスURL
  • ページ上の音声・映像コンテンツ
  • いかなる個人情報
  • 利用統計・アナリティクスデータ

ローカルに保存されるデータ

SpeedSense は以下のデータを chrome.storage.local に保存します。保存先はお使いのデバイスのみで、外部に送信されることはありません。

  • 再生速度の設定（ドメインごと）
  • 無音検出のしきい値・遅延設定
  • キーボードショートカットの割り当て
  • 累積節約時間のカウンター
  • UI 言語設定（EN / JA）

これらのデータはデバイス外に送信されません。ポップアップ内のリセットボタン、または拡張機能の削除によっていつでも消去できます。

外部への通信

拡張機能のポップアップは表示用フォントを Google Fonts（fonts.googleapis.com）から読み込みます。このリクエストにユーザーデータは含まれません。外部通信を避けたい場合でも、拡張機能の主要機能には影響しません。

権限の説明

  • storage — 設定をローカルに保存するため
  • tabs — ポップアップとアクティブページ間の通信、およびサポートページを開くために使用
  • host_permissions (<all_urls>) — video 要素を持つ任意のページにスクリプトを注入するために必要。各ページのデータは読み取りも送信もしません
